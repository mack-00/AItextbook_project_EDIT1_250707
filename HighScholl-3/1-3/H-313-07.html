탐색 알고리즘의 장단점 비교

순차 탐색과 이진 탐색의 장점과 단점을 비교해 보자.
순차 탐색과 이진 탐색의 수행 시간 비교 결과
알고리즘 장점 단점
순차 탐색
  • 정렬되지 않은 데이터에서도 사용 가능함.
  • 구현이 간단하고 이해하기 쉬움.
  • 평균적으로 수행 시간이 O(n)로 길어질 가능성 높음.
  • 데이터가 많아질수록 비효율적임.
이진 탐색
  • 수행 시간이 적게 소요됨.
  • 데이터가 많아져도 효율적으로 수행함.
  • 반드시 정렬된 데이터가 필요함.
  • 구현이 순차 탐색보다 복잡함.
  • 데이터가 자주 추가되거나 삭제되면 정렬부터 다시 수행해야 해서 비효율적임.
