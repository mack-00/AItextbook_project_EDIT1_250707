기계학습

전통적인 프로그래밍과 기계학습의 차이

  전통적인 프로그래밍에서는 사람이 직접 규칙과 조건을 설정하지만, 기계학습에서는 컴퓨터가 충분한 데이터를 학습하여 스스로 규칙을 배운다. 따라서 새로운 문제가 생겨도 규칙을 새로 만들 필요가 없고, 성능을 향상하기 위해 더 많은 데이터를 학습하거나 모델을 개선할 수 있어 사람이 규칙을 만들기 어려운 복잡한 문제를 효율적으로 해결할 수 있다.
전통적인 프로그래밍과 기계학습의 차이
  • 전통적인 프로그래밍
  • 기계학습
  Python, C, Java 등과 같은 프로그래밍 언어를 사용하여 명확한 규칙(또는 알고리즘)에 따라 작성하는 일반적인 방법을 뜻한다.

  “만약 이메일 제목에 ‘이벤트’, ‘우승’이라는 단어가 있으면 이메일을 스팸으로 분류한다.”와 같은 규칙을 구현하려면 코드를 직접 작성해야 한다.
새로운 상황이 생기면 규칙을 바꾸거나 추가해야 한다.
  • 입력
    (규칙)

  • 프로그래밍
  • 정답
    (출력 결과)
입력
이메일의 제목, 내용, 발신자 정보 등
규칙
  • 사람이 미리 정의한 특정한 단어에 따라 분류
  • 만약 제목에 ‘이벤트’, ‘우승’의 단어가 있으면 스팸으로 분류
정답
스팸 또는 정상 이메일
  명확한 규칙(또는 알고리즘)을 사람이 작성하는 것이 어려운 경우 데이터를 통해 컴퓨터가 패턴을 학습하고 규칙을 도출해 낼 수 있다.

  기계학습은 많은 이메일을 분석하여 스팸 메일과 정상 메일의 규칙을 스스로 알아낸다. 이후 새로운 이메일에 이 규칙을 적용하여 스팸 여부를 스스로 판단할 수 있다.
  • 입력
    (정답)

  • 기계학습
  • 규칙
    (학습된 규칙)
입력
이메일의 제목, 내용, 발신자 정보 등
정답
입력되는 이메일에 대해 스팸인지 아닌지에 대한 레이블
규칙
‘당첨’, ‘무료’, ‘이벤트’, ‘우승’ 등의 단어가 스팸에 자주 나타나는 현상을 파악하여 해당 단어가 등장하면 스팸으로 처리
레이블
각 입력 데이터에 대해 원하는 출력값(정답) 또는 목푯값을 의미한다.
